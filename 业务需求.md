# 业务需求规格说明书 (Requirements Specification)

## 1. 项目背景
本系统旨在为集团工厂建立统一的设备管理平台 (EMS)，解决设备规模大、分布广、管理难度高的问题。

*   **设备规模**：约 50,000 台
*   **分布范围**：10+ 基地，40+ 工厂，100+ 车间
*   **用户规模**：10,000+ 人（含操作工、维修工、工程师、管理层）
*   **核心痛点**：点检造假（"云点检"）、维修响应慢、数据孤岛、并发访问压力大。

## 2. 用户角色与权限

| 角色 | 职责描述 | 权限范围 |
| :--- | :--- | :--- |
| **系统管理员** | 基础数据维护、人员权限配置、数据导入 | 全局配置权限 |
| **设备主管** | 查看报表、监控设备状态、分析 OEE | 所属工厂/基地的数据查看权限 |
| **设备工程师** | 制定点检/保养计划、审核维修记录、管理知识库 | 所属工厂的计划与审核权限 |
| **维修工** | 执行维修任务、二级保养、领取备件 | 接收派单、填写维修记录 |
| **操作工** | 执行日常点检、一级保养、发现故障报修 | 仅限操作相关设备 |

## 3. 核心功能模块

### 3.1 设备台账管理
*   **组织架构**：支持 基地 -> 工厂 -> 车间 的三级架构管理。
*   **设备档案**：管理设备基础信息、技术参数、所属位置。
*   **二维码管理**：每台设备拥有唯一二维码，用于现场扫码识别。
*   **批量导入**：支持通过 Excel/JSON 批量导入设备和关联关系。

### 3.2 点检管理 (Inspection)
*   **防伪执行**：必须现场扫描设备二维码才能开始点检，防止虚假执行。
*   **模板配置**：支持按设备类型（如 CNC、焊机）配置不同的点检项目。
*   **任务生成**：系统每日（或按周期）自动生成点检任务列表。
*   **异常处理**：点检发现 NG 项自动触发报修流程（逻辑关联）。

### 3.3 维修管理 (Repair)
*   **移动报修**：任何人员发现故障均可扫码报修，支持拍照上传。
*   **自动派单**：优先派给设备绑定的专属维修工，无绑定则推送到公共池或由工程师指派。
*   **维修闭环**：报修 -> 接单 -> 维修 -> 试机 -> 报修人确认 -> 工程师审核 -> 归档。
*   **备件关联**：维修过程中记录消耗的备件，自动扣减库存。

### 3.4 保养管理 (Maintenance)
*   **分级保养**：支持一级保养（操作工）、二级保养（维修工）、精度维护等不同级别。
*   **弹性周期**：支持设置保养周期（如月度/季度），并提供弹性执行窗口（如前后 3 天）。
*   **计划日历**：可视化展示保养计划。

### 3.5 备件管理 (Spare Parts)
*   **库存管理**：以工厂为维度的备件库存管理。
*   **领用记录**：必须关联工单进行领用，确保成本可追溯。

### 3.6 统计分析 (Analytics)
*   **核心指标**：自动计算 MTTR (平均修复时间)、MTBF (平均故障间隔)。
*   **执行率**：点检完成率、保养按期完成率。
*   **OEE**：展示设备综合效率（支持外部数据导入展示）。

### 3.7 知识库 (Knowledge Base)
*   **经验沉淀**：优秀的维修记录可一键“升级”为知识库条目。
*   **故障查询**：维修工在现场可根据故障现象搜索解决方案。

## 4. 非功能需求
*   **高并发支持**：必须支持早班开工期间（早间高峰）数千人同时并发提交点检数据。
*   **多端适配**：
    *   **PC 端**：管理配置、数据分析（Vue3 + Element Plus）。
    *   **移动端**：现场作业（Vue3 + Vant H5），集成于企业 App 或独立使用。
*   **部署要求**：Docker 容器化部署，支持云端弹性伸缩。
*   **数据安全**：账号密码认证，后续预留 SSO 接口。
