Scope · 能力边界

只推荐，不执行

能力边界

Agent 应该做

推荐 · 审计 · 分析

  • 基于手册和历史数据推荐保养方案
  • 检测可疑维修记录并标记异常
  • 提供工程与管理分析支持
  • 生成推荐草稿和审计报告
  • 给出有证据支撑的结论
Agent 不应该做

不直接执行业务变更

  • 不直接修改保养计划
  • 不关闭维修工单
  • 不绕过权限访问跨厂数据
  • 不替代 QR 扫码等反作弊机制
  • 不扣减备件库存或审批费用

Architecture Principle
"规则先行，
LLM 解释在后。"

Rules and metrics identify issues.
Structured analysis computes comparisons.
The LLM explains findings and writes recommendations.

Step 1
确定性规则识别异常和推荐候选
Step 2
结构化分析计算对比、异常评分和趋势
Step 3
LLM 解释发现、编写可读推荐
