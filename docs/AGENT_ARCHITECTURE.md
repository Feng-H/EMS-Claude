# EMS-Claude Agent 架构设计与运行机制

> 本文档详细描述了 EMS-Claude 智能运维 Agent 的核心架构、运行逻辑、核心组件及自我进化机制。

---

## 1. 整体架构概述

EMS-Claude 的 Agent 系统采用了**分层式、领域驱动且可自我进化**的架构设计。它不仅仅是一个对话机器人，而是深度集成在设备管理业务流中的核心引擎，实现了从“被动响应”到“主动预测”的转变。

### 1.1 核心分层
- **接入层 (API Controller)**：提供标准化的 REST API，支持 Web 端、移动端及外部 AI Agent 调用。
- **编排层 (Service Engine)**：`AgentService` 负责多轮对话管理、意图识别、技能执行及异步反思。
- **能力层 (Analyzers & Tools)**：包含专业领域的算法分析器（RUL/TCO）和执行具体任务的工具集。
- **数据层 (Repository)**：支持结构化业务数据（PostgreSQL）与非结构化知识（RAG 向量检索）的双模存储。

---

## 2. 双路径运行模型 (Dual-Path Model)

Agent 系统根据任务性质，通过两条不同的路径运行，平衡了“确定性逻辑”与“生成式理解”。

| 路径 | 触发方式 | 运行机制 | 特点 |
| :--- | :--- | :--- | :--- |
| **对话式交互 (Assistant)** | 自然语言 Chat | **ReAct 循环**：意图识别 -> 技能匹配 -> 工具调用 -> 证据收集 -> LLM 总结。 | 灵活性高，适合故障排查和决策支持。 |
| **结构化分析 (Audit)** | 系统表单/自动触发 | **规则主导**：Analyzer 执行硬核算法逻辑 -> LLM 仅负责语义化报告生成。 | 确定性强，适合合规审计、月度分析。 |

---

## 3. 核心组件构成

### 3.1 编排引擎 (Agent Service)
- **意图识别**：利用 LLM 将用户模糊的需求转换为具体的目标。
- **执行闭环**：在 `ExecuteSkill` 方法中，Agent 会不断判断当前信息是否完整，并自主决定下一步调用的工具。

### 3.2 预测性分析引擎 (Predictive Analyzer)
内置 4 大核心分析维度：
- **RUL 预测**：计算设备剩余健康寿命。
- **TCO 计算**：全生命周期总成本（维修费 + 停机损失 + 折旧）。
- **亚健康检测**：识别“频发微停”、“保养无效”等隐性风险。
- **退役评估**：基于财务和健康分提供设备更新建议。

### 3.3 统一工具注册表 (Tool Registry)
所有 Agent 能力（读取档案、查询故障、搜索知识等）均封装为标准工具：
- **只读工具**：收集证据。
- **写入工具**：执行业务操作（如创建报修单、入库备件）。
- **权限校验**：每个工具在执行前都会经过 `Policy` 层的工厂隔离检查。

---

## 4. 技能 (Skill) 系统与 SOP

“技能”是 Agent 的核心能力载体，定义了一系列有序的工具调用步骤（SOP）。

- **定义方式**：通过 JSON 描述步骤。例如：`[Step 1: 获取档案, Step 2: 分析 MTBF, Step 3: RUL 预测]`。
- **动态匹配**：系统根据用户消息的内容、所属工厂、设备类型，动态匹配最适用的“技能”。
- **证据溯源**：Agent 每次调用工具产生的结果都会被标记为 `EvidenceItem`，确保结论“有据可查”。

---

## 5. 自我进化机制 (Reflect & Learn)

系统通过异步闭环实现能力的持续进化：

1. **对话归档**：对话结束后触发异步协程。
2. **反思学习**：LLM 分析对话记录，识别出新的故障排查套路。
3. **知识/技能提炼**：
   - **知识 (Knowledge)**：提取故障根因，存入 RAG 库。
   - **技能 (Skill)**：提炼出通用的 SOP 步骤，创建技能草稿。
4. **人工审核**：管理员在后台审核这些由 AI 提炼的成果，确认后正式激活。
5. **能力反哺**：激活后的知识/技能将在下一次服务中被优先召回。

---

## 6. 外部集成协议 (Tool Protocol)

EMS-Claude 向外部系统（如 OpenClaw、Claude Desktop）暴露了标准化的协议：

- **工具发现**：外部 Agent 可动态获取 EMS 的能力清单。
- **API Key 认证**：提供细粒度的权限控制和频率限制。
- **Webhook 推送**：支持主动将预警事件推送到外部平台。

---

## 7. 安全与数据隔离

- **工厂级隔离**：Agent 内部严格遵守工厂 ID 过滤，防止跨厂区数据泄露。
- **证据链可见**：向用户展示分析过程中的每一个步骤，降低 LLM 幻觉带来的风险。
- **双重审计**：记录 Agent 的每一次 Token 消耗和响应时间，用于成本控制和审计。
